Questions

  1. You have to have ng-app in your html and an ng-module in you js that links them.
  2. A controller is what you write the functions that you want to push into the html. The controller can take from both the html and factory.
  3. ng-model is like an id that you call in you controller with (vm.). ng-models have whatever name you decide to give them.
  4. Angular directive attributes take either expressions or interpolation markup with embedded expressions. It is considered bad practice to embed interpolation markup inside an expression.
  5. $scope is what you use to grap big information like an array of things.
  6. A dependency injection is what you have to inject to tell your files that you are going to use what you are gonna use. for example if you are using $http, $q, $scope, or and other things you must put them in your dependencies.
  7. Two-way binding is when you bind two things together so if you change them in one place they will change in the other place.
  8. 'controller as' is how you define how you are going to call things. (vm.) is used for one but you can make it anything. Example: controllerAs: 'Tristan'. Also you have to clarify to your controller what you are using, so you would put the name of your controller and then 'as vm' or 'as Tristan' whatever you are using.
  9. It's a framework of clean code writing.
  10. Factory is mostly preferable in all cases. It can be used when you have constructor function which needs to be instantiated in different controllers. Service is a kind of Singleton Object. The Object return from Service will be same for all controller.
